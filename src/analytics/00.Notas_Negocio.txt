Vamos começar a explorar alguns indicadores a partir da base de dados de loyalty system. Essa base
contém as transções que acontecem na plataforma da twitch. Os usuários ativos são medidos através
dos pontos obtidos na plataforma.

O publico alvo das análises é o usuário ativo nas redes, medido através dos pontos conquistados nas
seguintes situações:
    
    1. 1 msg no chat : 1 pto
    2. Enviando um !presente : 5 ptos
    3. Alcançando 5 dias seguidos de !presente : 100 ptos (aditivos por cada streak)
        - streak : !presente em 5 dias seguidos

Métricas de Negócio :

    1. DAU (Daily Active Users) : Usuários ativos no dia, medido pela quantidade de pontos

        - É bastante sensível ao capturar alta granularidade
        - Nessa granularidade de abertura é possível perceber que após os cursos há
        uma elevação da media diaria de DAU
        - Alavancas do DAU :

            - Trazer mais gente pra live/mar aberto (aumentar o topo do funil)
            - Aumentar a permanencia dos usuários ativos

    2. MAU (Monthly Active Users) : Usuários ativos no mês, medido pelo mesmo critério do DAU

        - É menos sensível que o DAU pois agrega o valor no mês
        - Deve ser calculada em janelas moveis de 28 dias (4 semanas completas, o que garante que
        estamos olhando sempre 4 AMOSTRAS DE CADA DIA DA SEMANA)
        - Cada dia reflete a janela de 28 dias para tras, então quedas nessa métrica são esperadas
        quando uma janela de um pico de DAU se extingue sem manutenção dos usuários ativos
        
Ciclo de Vida do usuário

    A ideia aqui é enxergar o tempo de vida do usuário a partir do primeiro momento em que o Cliente
    foi detectado na base de dados. Vamos considerar algumas outras métricas para compor nossos 
    grupos de usuários.

    Permite fazer a gestão do cliente de acordo com o momento dele em nosso ecossistema

    (a) Idade na base : Tempo desde a primeira transação (ao ultrapassar o status de Novo o Cliente
    não consegue mais voltar ao status anterior)
    
        - Novos (0-7 dias)
        - Antigos (7+ d)
    
    (b) Recência : Quantidade de dias desde a última interação (os status de recência podem ser 
    modificados ao longo do tempo, então o mesmo usuario pode transitar mais de uma vez em cada
    uma dessas classificações)

        Movimentos a montante: 
            
            - Fiel/Engajado (0-7 d) 
            - Turista (7-15 d) 
            - Desencantado (15-28d) 
            - Churn/Zumbi (28+ d)
        
        Movimentos a jusante

            - Turista -> Fiel : Loop irrelevante pro modelo
            - Desencantado -> Fiel : Reconquistado 
            - Churn -> Fiel : Renascido
    
    A curva de recência pode ser CALCULADA USANDO UMA CURVA DE SOBREVIVÊNCIA
    
    Fontes de dados para calculo das metricas

        (a) Idade na base : data da primeira transação
        (b) Ultima transação : calcular a recência do usuário (movimento recência a montante)
        (c) Penultima transação : calcular a recência o usuário (movimento recência a jusante?)

Frequência e Valor

    Essas dimensões estimam a INTENSIDADE do engajamento do cliente com a marca, por exemplo:

        - O cliente engaja em um ou mais produtos?
        - O cliente compra ou engaja quantas vezes ao mes?
        - O cliente interage com alto ou baixo valor?
    
    A Frequência pode ser calculada em DIAS ou quantidade de TRANSAÇÕES

        - A frequencia deve ser calculada dentro do espaço de base ativa (ou no caso de ecomm pode-se
        usar o período do MAU como referencia). Olhar para alem disso distorce a análise ao considerar
        um comportamento fora da metrica alvo ou ate mesmo fora do que faz sentido analisar, ja que
        estamos olhanndo a intensidade de relacionamento DOS QUE SE RELACIONAM

    O Valor pode ser calculado no ACUMULADO ou parametrizado por TRANSAÇÃO
